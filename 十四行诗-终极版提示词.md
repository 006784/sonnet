项目名：Sonnet（十四行诗）

- Rhythm：有节奏，不拥挤
- Warmth：有温度，不冷硬
- Intelligence：聪明但克制

一句话目标：
让“每一笔记录，都像生活里被认真写下的一行诗”。

================================================================
四、统一设计系统（这是全项目唯一视觉规范）
================================================================

设计哲学：
1. 留白即呼吸。页面不要塞满，元素之间要有停顿感。
2. 克制即高级。减少廉价阴影、渐变滥用、花哨装饰。
3. 节奏即韵律。间距、字号、圆角、动画要形成稳定节奏。
4. 墨色有层次。标题、金额、备注像不同笔锋写在纸上。
5. 智能感要内敛。AI 是辅助，不是炫技入口。

4.2 字体系统（标题有文学感，金额有秩序感）
----------------------------------------------------------------

请把以下规范收敛到 `Design/SonnetTypography.swift`：

- 标题：优先系统 `serif` 设计；如果仓库已接入更合适的中文衬线字体，可复用
- 正文：系统默认无衬线
- 金额：使用 `monospacedDigit()` 或等宽数字特性

建议层级：
- `titlePage`: 20, semibold, serif
- `titleSection`: 16, semibold
- `titleCard`: 15, medium
- `amountHero`: 36, semibold, negative tracking, monospaced digit
- `amountLarge`: 24, semibold, negative tracking, monospaced digit
- `amountBody`: 15, medium, monospaced digit
- `body`: 14, regular, line height ~22
- `caption`: 11-12, regular
- `label`: 10, medium

规则：
- 页面标题与日期标题可带轻微衬线气质
- 金额始终视觉优先级最高
- 备注和辅助信息必须轻，但不能淡到看不见

4.4 动效系统（柔和，不机械）
----------------------------------------------------------------

请把统一动画常量放入 `Design/SonnetMotion.swift`：

- 页面进入：淡入 + 轻微上移
- 卡片出现：stagger 级联
- 按钮点击：轻微缩放
- 数字变化：滚动或数值渐变
- FAB 展开：弹性展开
- 保存成功：勾号浮现 + 轻微回弹 + 淡出

推荐：
- 普通弹性：`response 0.35~0.45`
- 柔和展开：`response 0.5~0.65`
- 轻快按钮：`response 0.25~0.35`

不要：
- 廉价大幅旋转
- 过长延迟
- 每个元素都动

----------------------------------------------------------------

在 `Utils/HapticManager.swift` 或现有触觉封装中统一提供：

- 警告：删除、预算超限、错误提示

所有关键动作都应有轻微触觉反馈，但不要泛滥。

七、AI 能力（必须接入，但要克制）
================================================================

目标：
让用户“更快录入、更少手动输入、得到有价值的总结”。

AI 能力包括：
1. 小票 OCR 解析
2. 账单截图批量解析
3. 语音记账解析
4. 智能分类推荐
5. 月度消费洞察

服务层建议：
- `AIService.swift`：统一接口
- `OpenRouterClient.swift`：网络实现
- `OCRService.swift`：Vision 文字识别
- `SpeechService.swift`：语音识别

OpenRouter 规则：
- 默认模型：轻量、快、便宜
- 允许切换模型：Claude Haiku / Gemini Flash / DeepSeek / Qwen
- API Key 不允许硬编码
- 优先从 `Secrets.plist`、Keychain 或本地安全配置读取

请求风格：
- 统一使用 system + user prompt
- 明确要求只返回 JSON
- 低 temperature，保证结构稳定

统一结果结构：

```swift
struct ParsedRecordResult: Codable {
    let amount: Double?
    let merchant: String?
    let category: String?
    let date: String?
    let note: String?
    let type: Int?
    let confidence: Double

```

AI 交互原则：
- 当置信度不足时，不要自动直接入账，先让用户确认
- 识别失败要给出明确重试路径
- 所有 AI 结果必须可编辑
- AI 洞察文案要简洁、有信息量，不要空洞鸡汤

----------------------------------------------------------------
8.1 首页 Home
----------------------------------------------------------------

目标：
首页像一本当天翻开的诗集封面，承载“本月状态 + 每日记录”。

结构从上到下：
1. 顶部标题栏
2. 月度摘要卡片
3. 按日期分组的记录列表
4. 底部导航
5. 可展开 FAB

视觉要求：
- 背景：`paper`
- 水平边距：20
- 顶部标题保持克制，不堆按钮
- 摘要卡片使用 Ink 渐变或高级纸感材质，不要廉价高饱和色块

摘要卡片内容：
- 当前月份
- 当前账本名称
- 支出 / 收入 / 结余
- 可选预算进度
- 数字需要有滚动或淡入变化

记录列表：
- 按天分组
- 分组标题显示今天 / 昨天 / 日期
- 每条记录包含：分类图标、分类名、备注、金额
- 支出用 `vermilion`，收入用 `jade`
- 行高统一、分隔线克制

FAB：
- 默认悬浮在右下
- 点击展开 3 个动作：
  - 记一笔
  - 拍照识别
  - 语音记账

8.2 记账页 Record
----------------------------------------------------------------

目标：
记账页像一张正在书写的纸，输入流程流畅到几乎没有阻力。

结构：
1. 顶部标题
2. 收入 / 支出切换
3. 金额输入区
4. 备注与日期
5. 分类网格
6. 自定义数字键盘
7. 保存成功反馈

要求：
- 金额是主视觉中心
- 数字键盘和分类网格样式统一，不要系统默认味太重
- 支出 / 收入切换要明显但克制
- 保存后给出短暂成功状态，再返回或清空继续输入
- 支持添加自定义分类

8.3 统计页 Statistics
----------------------------------------------------------------

目标：
统计页不是“后台报表”，而是清晰、安静、有信息密度的阅读型页面。

内容：
- 月份切换
- 支出 / 收入切换
- 总额摘要
- 分类占比图（甜甜圈图或饼图）
- 分类排行列表
- 月度趋势
- AI 洞察卡片

要求：
- 图表样式与全局设计系统统一
- 不要默认工程化图表皮肤
- 空态时给出优雅说明
- AI 洞察卡片在关闭 AI 或无数据时优雅隐藏

内容：
